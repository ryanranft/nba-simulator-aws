**File:** `scripts/autonomous/autonomous_loop.service`

**Features:**
- Auto-start on system boot

- Resource limits (memory, CPU)
- Proper logging and error handling

## The Complete Autonomous Cycle

### Continuous Operation

**1. Reconciliation (Every Hour)**
```
Daemon wakes up
  â†’ Scan S3 bucket (AWS Inventory)
  â†’ Analyze coverage (HAVE vs SHOULD)
  â†’ Detect gaps (4-level priorities)
  â†’ Generate task queue (inventory/gaps.json)
  â†’ Sleep for 1 hour
  â†’ Repeat
```

**2. Task Monitoring (Every 30 Seconds)**
```
Check inventory/gaps.json
  â†’ Count available tasks
  â†’ If tasks >= threshold (default: 1):
      â†’ Trigger orchestrator
  â†’ Else:
      â†’ Continue monitoring
```

**3. Orchestration (On-Demand)**
```
Orchestrator starts
  â†’ Read task queue
  â†’ Execute scrapers by priority (CRITICAL â†’ LOW)
  â†’ Track progress
  â†’ Update inventory
  â†’ Trigger new reconciliation
  â†’ Exit (wait for next trigger)
```

**4. Health Monitoring (Continuous)**
```
Background thread (every 60 seconds)
  â†’ Check component health
  â†’ Log warnings for unhealthy components
  â†’ Restart failed components when possible
  â†’ Expose metrics via HTTP
```
